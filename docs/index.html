Something Fruity User Docs

User documentation for the web-based game 'Something Fruity'.

version 1.0.0

GET /accounts/student/{studentId}

Used to get an account by the external ID of the student associated with it. See the request sample below. 
                    					
GET /accounts/student/c6666666 HTTP/1.1
Content-Type: application/json;charset=UTF-8
Host: localhost:8080
The following parameter must be provided:
Parameter Description

studentId

The external ID of the student associated with this account.
The API sends back a response similar to the sample below. 
                        				
HTTP/1.1 200 OK
Content-Type: application/hal+json
Content-Length: 475
{
   "id":5,
   "studentId":"c6666666",
   "hasOutstandingBalance":false,
   "_links":{
      "self":{
         "href":"http://localhost:8080/accounts/student/c6666666"
      },
      "accounts":{
         "href":"http://localhost:8080/accounts"
      }
   }
}

The following fields are included in each account returned in the response:

Path Type Description

id

Number

The internal ID of the account.

studentId

String

The external student ID.

hasOutstandingBalance

Boolean

Whether the account has an outstanding balance.

The following links are included in the response:

Relation Description

self

Link to this resource.

accounts

Link to all accounts.

POST /accounts

Used to create a new account for a student. Send a POST request to the /accounts endpoint with a JSON body containing the external student ID. See the request sample below. 
                    					
POST /accounts HTTP/1.1
Content-Type: application/json;charset=UTF-8
Content-Length: 25
Host: localhost:8080

{"studentId": "c3429928"}

studentId

String

The external student ID.

GET /invoices/reference/{reference}

Used to get an invoice by its unique external reference. See the request sample below. 
                    					
GET /invoices/reference/BAX5SPP5 HTTP/1.1
Content-Type: application/json;charset=UTF-8
Host: localhost:8080
The following parameter must be provided:
Parameter Description

reference

The external reference of the invoice to return.
The API sends back a response similar to the sample below. 
                        				
HTTP/1.1 200 OK
Content-Type: application/hal+json
Content-Length: 397
{
   "id":10,
   "reference":"BAX5SPP5",
   "amount":10.5,
   "dueDate":"2021-12-25",
   "type":"TUITION_FEES",
   "status":"OUTSTANDING",
   "studentId":"c6666666",
   "_links":{
      "self":{
         "href":"http://localhost:8080/invoices/reference/BAX5SPP5"
      },
      "invoices":{
         "href":"http://localhost:8080/invoices"
      },
      "cancel":{
         "href":"http://localhost:8080/invoices/BAX5SPP5/cancel"
      },
      "pay":{
         "href":"http://localhost:8080/invoices/BAX5SPP5/pay"
      }
   }
}

The following fields are included in each account returned in the response:

Path Type Description

id

Number

The internal ID of the invoice.

reference

String

The external invoice reference used to pay an invoice via the Payment Portal.

amount

Number

The invoice amount.

dueDate

String

The date the invoice must be paid by.

type

String

The type of invoice. Types currently enabled are: LIBRARY_FINE and TUITION_FEES.

status

String

The status of the invoice: OUTSTANDING, PAID or CANCELLED.

studentId

String

The external ID of the student responsible for paying this invoice.

The following links are included in the response:

Relation Description

self

Link to this resource.

invoices

Link to all invoices.

cancel

Link to cancel this invoice.

pay

Link to pay this invoice.

POST /invoices

Used to create a new invoice. Send a POST request to the /invoices endpoint with a JSON body containing the required fields. See the request sample below. 
                    					
POST /invoices HTTP/1.1
Content-Type: application/json;charset=UTF-8
Content-Length: 102
Host: localhost:8080
{
   "amount":15.00,
   "dueDate":"2021-11-06",
   "type":"LIBRARY_FINE",
   "account":{
      "studentId":"c6666666"
   }
}
The following fields must be provided when creating a new invoice:
Path Type Description

amount

Number

The invoice amount.

dueDate

String

The date the invoice must be paid by (yyyy-MM-dd).

type

String

The type of invoice. Types currently enabled are: LIBRARY_FINE and TUITION_FEES.

account.studentId

String

The external ID of the student responsible for paying this invoice.

Privacy Policy

Something Fruity is part of the Something Fruity Group which includes Something Fruity International and Something Fruity Direct. This privacy policy will explain how our organisation uses the personal data we collect from you when you use our website.
What data do we collect? Something Fruity collects the following data:
  •   Personal identification information (Name, email address, phone number, etc.)
  •   Number of games played, login dates and length of time of sessions.
How do we collect your data? You directly provide Something Fruity with most of the data we collect. We collect data and process data when you:
  •   Register online to create a new user account.
  •   Login to your account.
  •   Play any of our games while you are logged in.
How will we use your data? Something Fruity collects your data so that we can:
  •   Manage your account.
  •   Email you with special offers on other products and services we think you might like.
How do we store your data? Something Fruity securely stores your data in a database in the cloud at AWS.
Something Fruity will keep your account details for five years after a period of inactivity. Once this time period has expired, we will delete your data.
Marketing Something Fruity collects the following data:
  •   Personal identification information (Name, email address, phone number, etc.)
  •   Number of games played, login dates and length of time of sessions.
What are your data protection rights? Something Fruity would like to make sure you are fully aware of all of your data protection rights. Every user is entitled to the following:
  • The right to access – You have the right to request Something Fruity for copies of your personal data. We may charge you a small fee for this service.
  • The right to rectification – You have the right to request that Something Fruity correct any information you believe is inaccurate. You also have the right to request Something Fruity to complete the information you believe is incomplete.
  • The right to erasure – You have the right to request that Something Fruity erase your personal data. You can delete your account and your gameplay history from your account page. Please be advised that this process is irreversible.
  • The right to restrict processing – You have the right to request that Something Fruity restrict the processing of your personal data, under certain conditions.
  • The right to object to processing – You have the right to object to Something Fruity’s processing of your personal data, under certain conditions.
  • The right to data portability – You have the right to request that Something Fruity transfer the data that we have collected to another organisation, or directly to you, under certain conditions.

    • If you make a request, we have one month to respond to you. If you would like to exercise any of these rights, please contact us
      by email:  something-fruity@gmail.com
      by mail:   Something Fruity, University of Essex Online.
Changes to our privacy policy Something Fruity keeps its privacy policy under regular review and places any updates on this web page. This privacy policy was last updated on 27 January 2022.
How to contact us If you have any questions about Something Fruity’s privacy policy, the data we hold on you, or you would like to exercise one of your data protection rights, please do not hesitate to contact us
